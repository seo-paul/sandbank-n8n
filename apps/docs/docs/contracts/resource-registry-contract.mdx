# Resource Registry Contract

## Purpose
- Definiert die kuratierte Ressourcenbasis fuer Retrieval, Acceptance und Usage.
- Verhindert, dass Laufzeitlogik nur auf groben Domainmustern basiert.

## Required Fields
- `version`
- `registry_name`
- `default_resource_profiles`
- `publishers`

## Default Resource Profiles
- Key = `resource_class`
- Pflichtfelder pro Klasse:
  - `source_type`
  - `source_tier`
  - `authority`
  - `evidence_strength_score`
  - `citation_readiness_score`
  - `commercial_bias_score`
  - `default_usage`

## Publisher Entries
- `resource_id`
- `publisher`
- `domain_patterns`
- `resource_class`
- `allowed_workflows`
- `allowed_usage`
- `manual_review_required`
- `bias_flags`
- `notes`
- optional: `topic_keywords_any`

## Resource Classes
- `official_product_or_platform_docs`
- `official_standards_or_regulation`
- `first_party_original_data`
- `industry_benchmark_or_survey`
- `operator_case_study`
- `topic_specific_research`
- `community_signal`
- `general_media`
- `vendor_content`
- `low_value_aggregator`

## Allowed Usage Modes
- `background`
- `fact`
- `quote`
- `comparison`
- `counterpoint`
- `example`

## Freshness Rules
- Freshness wird nicht im Registry-Eintrag fix gespeichert, sondern in der Pipeline aus `published_at` berechnet.
- Registry und Policy duerfen workflow-spezifische Mindestschwellen setzen.

## Review Flags
- `manual_review_required: true` bedeutet:
  - Quelle darf nicht stillschweigend als starke Evidenz behandelt werden
  - Reviewer muss Themenfit, Bias und konkrete Nutzung pruefen

## Example Entry
```json
{
  "resource_id": "broad-science-review-only",
  "publisher": "Broad Science Publishers",
  "domain_patterns": ["nature.com", "springer.com", "sciencedirect.com"],
  "resource_class": "topic_specific_research",
  "allowed_workflows": ["social", "bi-guide"],
  "allowed_usage": ["background", "counterpoint"],
  "manual_review_required": true,
  "topic_keywords_any": ["analytics", "business intelligence", "dashboard"],
  "bias_flags": ["generic_science_domain_not_default_bi_source"],
  "notes": "Nur bei direktem BI-/Analytics-Fit nutzen."
}
```
