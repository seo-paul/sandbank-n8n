# Contracts Overview

## Scope
Die Vertragsflaeche definiert Daten-, Qualitäts- und Governance-Regeln zwischen Workflow-Stages, Persistenz und Obsidian-Ausgabe.

## Contract Family
- Stage Contracts (Social + BI Guide): schema-basierte Übergaben zwischen Research-, Content-, Review- und Persistenzstufen.
- Registry/Policy Contracts: Ressourcenklassifizierung, Freshness-Regeln und Usage-Modes.
- BI Guide Spezialvertraege: Opportunity, Source Snapshot, Article Package, Refresh.

## Breaking-Change Policy
Als breaking gelten mindestens:
- Feldentfernung in `required`
- Enum-Verschärfung ohne Migrationspfad
- Semantikwechsel bei stabilen Schluesselfeldern

Empfohlener Ablauf:
1. Schema-Aenderung in Obsidian/SSOT vorbereiten.
2. Mirror via Pull/Sync angleichen.
3. Build/Validation laufen lassen.
4. Erst dann Workflow-Rollout.
